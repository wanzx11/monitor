运维监控系统——Java 后端工程师工作内容总结
作为 Java 后端工程师，在该运维监控系统项目中的主要工作内容包括：

1. 服务端开发
Spring Boot 3 框架搭建：基于 Spring Boot 3 进行后端服务的搭建，配置基础依赖，优化项目结构。

客户端注册管理：

采用 JSON 存储 方式管理客户端注册信息。

设计 客户端注册流程，提供 REST API 让客户端进行注册、更新、注销等操作。

使用 Redis 存储注册/重置验证码，控制验证码有效期。

监控数据接收与处理：

设计 WebSocket 通信机制，使前端能够实时操作 Shell 进行服务器管理。

处理 客户端上报的监控数据，包括 CPU、内存、磁盘等硬件信息。

采用 InfluxDB 作为时序数据库，提高监控历史数据的存储和查询效率。

采用 Redis 缓存机制，优化监控数据的查询，提高系统响应速度。

整理监控数据，生成时间段数据集，以便前端折线图展示。

定时任务调度：

采用 Spring Quartz 实现定时任务，定期清理过期数据，维护系统稳定性。

2. 客户端开发
基于 Oshi 框架 实现跨平台硬件信息采集，包括 CPU、内存、磁盘、网络等实时数据读取。

多账户权限管理：

采用 Spring Security + JWT 进行身份认证和权限校验。

允许不同服务器由不同账户管理，确保访问控制的安全性。

3. 系统安全与优化
权限管理：

采用 Spring Security 进行权限控制，手动整合 JWT 方案，确保用户身份验证安全可靠。

消息队列优化：

采用 RabbitMQ 处理短信验证码发送任务，减少接口调用压力，提高短信发送效率。

数据结构优化：

视图层对象（DTO）与数据层对象（Entity）分离，利用 反射工具方法 实现快速转换。

全局异常处理：

统一采用 JSON 格式 返回错误信息，便于前端解析和处理。

跨域问题：

采用 过滤器 手动处理跨域请求，确保前后端通信正常。
